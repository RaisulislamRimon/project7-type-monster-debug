localStorage and sessionStorage are both ways to store key or value pairs for a particular website/domain for browsers. The values stored may only be strings in both localStorage and sessionStorage. localStorage is limited to about 5MB and can contain only strings. SessionStorage is used for storing data on the client side. Maximum limit of data saving in SessionStorage is about 5 MB. SessionStorage is cleared when the browser is closed. localStorage is cleared when the user clears the browser cache.

There are two types of scopes in javascript. They are: Local and Global scope. Local scope is the scope of a variable that is defined inside a function. Global scope is the scope of a variable that is defined outside a function. Local scope variables are not accessible from outside the function. Global scope variables are accessible from anywhere in the code. If we declare a variable 'y' inside a function 'something.' The scope of that variable remains inside that function and it can't be used outside of that function. This is called local scope. If we declare a variable 'y' outside a function 'something.' The scope of that variable remains outside that function and it can be used anywhere in the code. This is called global scope.

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks. The Event Loop has one simple job to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.

There are 8 ways to get undefined in javascript. They are given below:
1. variable that is not initialized will give undefined
2. function with no return
3. parameter that is not passed will be undefined
4. if return has nothing on the right side will return undefined
5. property that doesn't exists on an object will give you undefined
6. accessing array elements outside of the index range
7. deleting an element inside an array
8. set a value directly to undefined
